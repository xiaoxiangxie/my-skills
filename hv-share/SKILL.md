# hv-share: 一图系列生成器

输入任意主题（城市、公司、概念、人物、产品），输出一套可分享的 HTML 卡片 + 浏览器内切换查看器 + 一键下载 PNG。

## 输入

- **城市名**：自动触发 ljg-travel 研究，叠加「参观路线」Tab
- **非城市词**（公司/产品/概念/人物等）：触发 hv-analysis 横纵分析

## 输出

`{output_dir}/index.html` — 浏览器内 tab 切换查看器  
每张卡片可独立下载 PNG（2x 精度，2160px 宽）

## 工作流程

### 步骤 1：判断主题类型

```
城市？→ 步骤 2a（ljg-travel）
非城市？→ 步骤 2b（hv-analysis）
```

### 步骤 2a：城市主题

1. 调用 `ljg-travel` skill，获取旅行研究成果（博物馆、古建、路线）
2. 解析 org 文档提取内容
3. 生成 6 张卡片：

| Tab | 主题 | 风格 |
|-----|------|------|
| 封面 | 城市概览封面 | 深蓝+金色，简洁大气 |
| A | 经济实力 | 深蓝+金色大数据面板 |
| B | 产业实力 | 墨绿+翡翠产业矩阵 |
| C | 历史底蕴 | 紫+金时间轴 |
| D | 文旅吸引力 | 深红+橙景点三栏 |
| E | 人口与城市 | 深蓝+天蓝双栏 |
| F | 参观路线 | 暖棕大字报（ljg-travel数据）|

### 步骤 2b：非城市主题（hv-analysis）

1. 调用 `hv-analysis` skill，获取横纵分析报告
2. 解析 Markdown 报告，提炼：
   - 核心数据（营收/用户数/市场份额等）
   - 发展时间线（关键节点）
   - 竞争优势/劣势
   - 竞品对比
   - 未来判断
3. 生成 5 张卡片（封面 + 4 内容卡）：

| Tab | 主题 | 风格 |
|-----|------|------|
| 封面 | 作品概览封面 | 对应主题主色，简洁大气有吸引力 |
| A | 核心指标 | 对应主题的核心数据展示 |
| B | 发展时间线 | 纵向历史节点 |
| C | 竞争格局 | 横向竞品对比 |
| D | 未来判断 | 趋势与洞察 |

4. **Tab E（文案页）**：调用 `dbs-content` 生成标题和话题文案，调用 `dbs-ai-check` 去 AI 味。该页**不生成下载按钮**，仅展示可复制文本供用户直接使用。

### 步骤 3：渲染 index.html

每张卡片内容作为隐藏 `<div data-tab="X">` 内嵌，JS 实现 tab 切换 + html2canvas 下载。

## HTML 卡片规范

### 尺寸（移动端优先）
- 卡片宽度：1080px（标准社交媒体图片尺寸）
- 高度：动态（min 1350px）
- html2canvas 缩放：2x（输出约 2160px 宽）
- **核心原则：小字体必须能在手机上看清，优先保证关键信息一目了然**

### 封面卡设计规范（第一个 Tab）
封面是整套卡片的第一印象，要求简洁大气、有吸引力：

**布局结构**：
- 标题居中靠上，超大字体（180px）
- 副标题在标题下方，概括核心主题
- 顶部可有 tagline 标签（如获奖信息、平台标签等）
- 中部可有一句核心引言或金句
- 底部放3-4个关键信息（作者、原著奖项、平台等）

**字体规范**：
- 主标题：Noto Serif SC 900，180px，字母间距 20px
- 副标题：36px，300 字重，字母间距 8px
- 标签/角标：28px
- 引言：36px，行距 1.7
- 底部信息标签：24px，信息值 32px

**配色**：
- 背景：深蓝渐变（#0d1b4b → #1a237e → #2d1b69）
- 强调色：金色 #fbbf24
- 可加径向渐变大圆球装饰

### 移动端可读性要求（核心约束）
生成卡片时必须遵守以下字体最小值，确保手机端分享后仍可读：

| 元素 | 最小字号 | 说明 |
|------|----------|------|
| 大标题 | 72px | 书名/城市名/核心主题 |
| 副标题 | 36px | 描述语/副题 |
| 标签/角标 | 28px | tag、badge 等 |
| 正文段落 | 28px | 主要描述文字 |
| 数据数字 | 64px | 核心 KPI 数据（加粗+强调色） |
| 数据标签 | 32px | 数字下方的说明文字 |
| 列表标题 | 32px | 卡片内各区块标题 |
| 列表描述 | 26px | 列表项的说明文字 |
| 时间线年份 | 32px | 时间线年份数字 |
| 引用文字 | 30px | 引用块文字 |

**其他移动端要求**：
- **行间距**：1.6-1.8 倍，确保呼吸感
- **信息密度**：宁可少放信息，不可让文字挤成一团
- **卡片布局**：单列为主，多列网格每列不窄于 300px
- **留白**：padding 最小 60px，内容区四周留足呼吸空间
- **数据卡片**：三列网格改为两列，避免列宽过窄

### 导航栏
- 固定顶部，毛玻璃效果
- 城市名作为标题
- 各 Tab 按钮带颜色指示线

### 下载按钮
- 位于卡片下方居中
- 颜色跟随当前 Tab 主题色
- 点击后生成 PNG 并自动下载

## 执行方式

在 Claude Code 中调用：

```
/hv-share [城市名或主题词]
```

示例：
```
/hv-share 长沙        → 城市主题，7 Tab（封面+6内容），含参观路线
/hv-share 三一重工    → 公司横纵分析，6 Tab（封面+4内容+文案）
/hv-share Hermès      → 品牌/公司横纵分析，6 Tab
```

## 错误处理

- ljg-travel 或 hv-analysis 失败 → 回退到基础信息卡（只用联网搜索的数据）
- 主题判断不明确 → 默认走 hv-analysis
- html2canvas 失败 → 提示浏览器兼容性
