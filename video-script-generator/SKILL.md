# Video Script Generator

将任意主题、资料或链接内容，提炼并重写为**适合剪映字幕朗读的逐行讲解文案**。

---

## 输出物（每次生成五个）

| 输出 | 用途 | 下游 |
|---|---|---|
| 1. 逐行字幕 | 剪映 TTS 朗读，导出 SRT | hyperframes 读取 SRT 控制时长 |
| 2. 风格指导 | hyperframes 制作视频的视觉规范 | DESIGN.md |
| 3. 文案 | 封面钩子、标题、话题、正文文案 | 运营分发（抖音/小红书） |
| 4. 内容诊断 | dbs-content + dbs-ai-check 产出 | 质量把控 |
| 5. 封面 HTML | 封面轮播预览，可导出 PNG | 剪映/Canva 使用 |

---

## 冥想类视频专用改造规则

当用户要求生成冥想课程的字幕文案时，**必须**引用以下改造规则文件：

> `references/meditation-hyperframe-rules.md`

该规则定义了：
- **跟练/实践类**：引导词完整保留，开场白/结束语精简（删除课程预告）
- **讲解类**：一个论点 = 一行，压缩结论删除例子

### 冥想类视频封面生成规则

**封面模板**：`references/cover-template.html`

**生成时机**：每次生成冥想课程内容时，必须同时生成 `cover.html`

**课程类型与 tag 对应**：
| 类型 | COURSE_TYPE |
|------|-------------|
| 先导片 | 先导片 |
| 讲解 | 讲解 |
| 跟练 | 跟练 |
| 小结 | 小结 |

**标题高亮建议**：
- 标题1：直接用 dbs-hook Top3 推荐 1，不加高亮
- 标题2/3：可根据内容决定是否用 `<span class="hl">...</span>` 高亮关键词

**标题中的高亮词选择**：
- 数字：`2亿人`、`120倍` → 高亮
- 情绪词：`压力`、`焦虑` → 高亮
- 动词：`知道`、`练习` → 根据语境决定

**封面生成后必须检查**：
- 标题文字是否超出容器宽度
- 三个尺寸是否都能正常显示
- 高亮 span 是否正确包裹关键词

### 课程名重命名规则（强制）

**核心原则：不得直接使用或衍生自原稿标题，必须从内容核心重新提炼。**

课程名 = 编号 + 重新命名的名称，从内容核心重新提炼，而非修改原文：

| 原稿名称 | 问题 | 重新命名 | 说明 |
|---------|------|---------|------|
| 「05-身体感受-正文」 | 直接截取原文 | `05-身体力行` | 强调行动实践 |
| 「01-01 冥想是什么？」 | 直接截取原文 | `01-起点` | 重新开始的含义 |

**重命名规则**：
- ✅ 可以使用原稿中的编号（如 `01`、`02`、`03`）
- ✅ 可以从内容核心重新提炼含义
- ✅ 可以用意象、比喻、行动导向等方式表达
- ✅ 必须保持与原稿内容一致的原意
- ❌ 不得直接截取或简单修改原稿标题
- ❌ 不得使用「第X课」形式
- ❌ 不得添加原稿中没有的含义

**命名思路**：
- 意象法：用比喻/意象表达（如「归零」「起点」）
- 行动法：用动词导向（如「做好这些准备」「动中冥想」）
- 问题法：用疑问引发好奇（如「你多久没有...」）

**判断标准**：如果用户问「这个课程名说的是这个内容吗」，回答必须是「是」。同时，课程名应该让不知道原稿内容的人也能理解这个课程在讲什么。

**必须删除的内容**：
- ❌ 课程预告：「下节课讲什么」「期待与您相见」「让我们一起爱上冥想」
- ❌ 课程承诺：「你一定要练到第二周」「14天系统学习」「一个月陪伴练习」
- ❌ 课程推广：「构建内心秩序」「找到平静和力量」「从零开始」
- ❌ 旁白词：「很好」「接下来」「Ok」「太棒了」「非常好」
- ❌ 结果想象：「想象完成后的成就感」
- ❌ 铺垫解释：「如果你感兴趣可以搜索XXX」
- ❌ 图片OCR注释
- ❌ 自我对话：「我念一句你跟念一句」

**压缩原则**：
- 原文一句句照搬 = 错误。必须提炼核心论点，压缩冗余
- 每行是一个独立论点，不是一句完整的原文复述
- 删除所有例子（"例如走到河边"等），只保留结论

### 正文文案改造规则（强制）

正文文案（用于抖音/小红书发布）**也必须遵循字幕改造规则**：
- 只做纯内容分享，不含任何课程产品推广信息
- 删除所有「几天」「几周」「课程」「系统学习」「陪伴」等词
- 删除「构建内心秩序」「找到平静和力量」等推广语
- 删除课程预告和课程承诺
- 基于原文压缩，不添加原意，不虚构

### 合规红线（强化）

以下词汇**禁止出现在任何输出物**中：
- 「14天」「21天」「一个月」等课程时长
- 「系统学习」「陪伴练习」「从零开始」
- 「构建内心秩序」「找到平静和力量」
- 「期待与您相见」「下节课见」「你一定要练到」
- 任何其他课程产品推广语

以上规则适用于：字幕、文案、内容诊断所有输出文件。

## 工作流

```
用户内容 → 字幕改造 → dbs-content 诊断 → dbs-ai-check 去AI味 → dbs-hook 优化开头 → dbs-xhs-title 生成标题 → 输出五个文件（含封面 HTML）
```

### Step 8：生成封面 HTML（冥想课程专用）

基于 `文案.md` 中的 **Top 3 推荐**，生成封面轮播 HTML：

**模板位置**：`references/cover-template.html`

**脚本位置**：`scripts/generate_covers.py`

**触发方式**：生成封面后，执行脚本生成 `cover.html`：
```bash
python3 scripts/generate_covers.py
```

**替换规则**：
| 占位符 | 说明 | 示例 |
|--------|------|------|
| `COURSE_NUM` | 课程编号（两位） | `01`、`02` |
| `COURSE_NAME` | 课程名称 | `归零`、`与脑共处` |
| `COURSE_TYPE` | 课程类型 | `先导片`、`讲解`、`跟练`、`小结` |
| `TITLE_1` | 推荐标题1 | 可用`<span class="hl">关键词</span>`包高亮 |
| `TITLE_2` | 推荐标题2 | 同上 |
| `TITLE_3` | 推荐标题3 | 同上 |

**封面配色**（已内置于模板）：
- 背景：#0B1B2B（深蓝）
- 高亮：#7FCDCD（青绿）
- 字体：Noto Serif SC

**高亮语法**：`<span class="hl">需要突出的词</span>`

### Step 1：生成逐行字幕

1. 接收用户提供的内容（主题、资料、链接等）
2. 提炼核心信息
3. 按逐行文案规则重写
4. 输出可直接粘贴到剪映的逐行文案

### Step 2：内容诊断（必须调用 dbs-content）

调用 `/dbs-content`：
- 诊断内容完整性、素材丰富度、话题范围
- 输出诊断报告

### Step 3：去AI味（必须调用 dbs-ai-check）

调用 `/dbs-ai-check`：
- 使文案更自然、口语化
- 去除AI生成的生硬表达

### Step 4：优化开头（必须调用 dbs-hook）

调用 `/dbs-hook`：
- 基于诊断后的内容生成封面钩子
- 生成 10-15 条优化方案
- 给出 Top 3 推荐

**重要**：dbs-hook 的 Top 3 推荐**必须同时应用到字幕开头**，不要只放在文案.md 里。取推荐 1 直接替换字幕前 1-3 行作为视频开头钩子。

### Step 5：生成标题（必须调用 dbs-xhs-title）

调用 `/dbs-xhs-title`：
- 从 75 个公式库中匹配最适合的公式
- 生成小红书标题（带公式编号）
- 生成话题标签

### Step 6：生成正文文案

基于以上产出，生成抖音/小红书发布的正文文案（约 50-100 字）：
- 补充标题没能说完的内容
- 引导评论互动
- 建立人设

### Step 7：生成风格指导

基于内容类型和主题，生成 hyperframes 所需的 DESIGN.md：

- **Style Prompt**：一句话描述视觉方向
- **Colors**：3-5 个 hex 值 + 角色说明
- **Typography**：1-2 种字体
- **Motion**：进入/退出动画建议
- **What NOT to Do**：3-5 个禁忌

## 逐行字幕规则

### 句长限制

- 最佳：10～18 字
- 最少：6 字
- 最大：不超过 25 字

### 节奏要求

- 每句朗读时长约 2～5 秒
- 语义完整，独立可读
- **使用标点断句，不使用空行分隔**

### 标点规则

必须使用自然停顿标点：`，` `。` `？` `！`

禁止：
- 无标点长句
- 连续多个逗号
- 复杂嵌套句
- 用空行代替标点断句

**正确格式**：
```
冥想能延长清醒状态，
不冥想上午就迟钝。

早起能一直保持高效，
到下午都清醒。
```

**错误格式**（用空行断句）：
```
冥想能延长清醒状态

不冥想上午就迟钝

早起能一直保持高效

到下午都清醒
```

### 语言风格

- 口语化，易理解，有节奏感
- 避免学术语言、书面长句、复杂逻辑嵌套
- 不用"首先/其次/最后"等强逻辑词

### 口语化规则（强制）

**每行字幕必须是一个完整通顺的句子。**

- ❌ 错误：「等有空再练不会去练」—— 不完整，读不通
- ✅ 正确：「等有空往往不练」—— 主谓宾完整，口语化
- ❌ 错误：「潜意识更快进入状态」—— 缺主语，不完整
- ✅ 正确：「潜意识更快帮你进入状态」—— 有主语有谓语
- ❌ 错误：「信息让注意力碎片化」—— 缺主语
- ✅ 正确：「信息让注意力变得碎片化」—— 主谓完整

**检查方法**：每行字幕必须能够：
1. 独立成句（有主语、有谓语）
2. 不依赖上下文字词就能理解
3. 朗读出来自然流畅，不像在念PPT

**常见错误模式**：
- 缺主语：「让注意力...」「导致杂念...」
- 不完整：「最有效的方法」「阻碍注意力」
- 书面语：「潜意识更快进入状态」
- 翻译腔：「进行冥想」「产生杂念」
- 不符合常理：「穿毯子」等不符合日常表达的用法

### 忠实原稿原则（强制）

**字幕改造必须遵循原稿，同时确保语句通顺符合常理。**

- ✅ 可以重新组织语言、压缩句子
- ✅ 可以删除例子和冗余描述
- ❌ 不得虚构、添加原稿中没有的信息
- ❌ 不得使用不符合常理的表达（如"穿毯子"）

**判断标准**：改造后的句子是否忠实于原稿含义，同时读起来自然通顺？

**典型错误修正记录**：
- 「下半身穿毯子或袜子」→「天冷时下肢要保暖，盖毯子或穿袜子都行」
- 原稿表述有误时（如"穿毯子"不合常理），应**重新组织语言**，不能机械保留原词

## 质量自检

输出前必须确认：
- 每行都是完整一句话
- 每行长度合理（6～25 字）
- 整体节奏均匀
- 没有特别长或特别短的句子

## 合规红线

**不得虚构内容，不得夸大事实，遵守法律法规。**

所有输出必须：
- 忠实于原素材文档，不添加原文中没有的信息
- 不夸大效果、功效、数据
- 不使用绝对化表述（"最"、"第一"、"保证"等）
- 涉及健康/医疗/金融等专业领域时，提示用户自行核实

### 版权规避规则（强制）

**核心原则：原稿来源网络，不得直接引用完整原文。**

所有输出物（字幕、文案、诊断）在使用原稿内容时：
- ✅ 可以提取核心论点、数据、概念重新表述
- ✅ 可以压缩原文保留结论、删除例子
- ✅ 必须标注来源：「内容来源：原稿」
- ❌ 不得完整复制原文段落
- ❌ 不得保留原稿中的人物姓名、机构名称（除非是「苹果公司」「《纽约时报》」等公众熟知品牌）
- ❌ 不得保留原稿中「李冉」「@李冉在冥想」等个人身份标识

**处理方式**：
- 原文的核心论点 → 用自己的话重新表达
- 原文的数据（如「2亿人」「120倍」）→ 可保留，因为数据本身不受版权保护，但需确认数据准确性
- 原文的比喻（如「大哥大玩3A游戏」）→ 可保留，因为是通用表达
- 原文的人物经历（如「2011年去尼泊尔闭关十天」）→ 删除，不使用

**判断标准**：如果用户问「这段话是我写的吗」，能回答「不是」的字，就不要用。
