# 内容提取方法

这个文件解决两件事：怎样把长文、访谈、播客逐字稿变成 note slides 的内容骨架；以及每一页应该长什么样。

主原则：先按材料顺序做笔记，再把笔记排成 slides。不要先写中心论点，再回头找材料填空。

## 页面写法

每页优先写成下列之一：

1. **原问题**：材料里真正被问到的那句话。
2. **原判断**：人物给出的判断或关键词。
3. **原例子**：用来证明判断的故事、数字、对照或类比。
4. **原顺序**：这一页接在上一篇材料节点之后自然出现。
5. **原余味**：结尾停在材料留下的问题、愿望或未完成处。

引号只用于严格原话和对话摘录。页面标题、概念词、关键词和压缩转述默认不加引号。不要为了强调给词语套引号。

不要写成：

1. 这篇文章先把什么摆出来，导读腔。
2. 这篇采访记录的是某种方法，评论腔。
3. 外界看到什么，人物解释为什么，站在材料外。
4. 本质 / 真正 / 最该带走的是某某，空泛拔高。
5. 不读原文也能写出的漂亮概括。

## 一、通读后的记录

通读材料后，只记录对生成有用的东西：

1. 基本信息：标题、作者、人物、时间、来源、正文长度。
2. 推进顺序：材料从哪里进入，依次转到哪些问题，最后停在哪里。
3. 原问题：访谈里真正推动回答的问题。
4. 原判断：人物给出的明确判断、立场、方法或概念。
5. 原例子：故事、数字、对照、类比、事件和产品。
6. 线索词：反复出现并改变含义的词。
7. 张力点：提问和回答之间、常识和回答之间、前后观点之间的拉扯。
8. 重话：能代表人物经验和判断的句子。

内部可以写摘要，但摘要不能直接成为页面标题。

## 二、候选页清单

按材料顺序列出所有候选页。每页五项：

1. **来源位置**：文章小标题、原问题、时间点或段落位置。
2. **页面重点**：观众要记住的一句话。
3. **支撑锚点**：原判断、原话、例子、数字或概念原词，至少其一。
4. **推荐布局**：引用、对比、表格、矩阵、时间线、大数字、段落拆解……
5. **自检**：不读原文能不能写出这一页？能则重写。

好节点像这样：

```text
来源：追觅的边界是什么
重点：边界不在制造，俞浩说自己也不知道终点
锚点：不知道边界在哪里，不过肯定不是制造
布局：居中原话加侧注
自检：通过，这是开场第一问的现场原话
```

不好的节点像这样：

```text
重点：这篇采访展示了一套面对不确定性的商业实验法
锚点：无
```

任何节点凑不出锚点，直接删掉。

## 三、访谈材料

访谈 slides 要保留现场感，但不搬运采访界面。

**保留**：

1. 真正推动回答的原问题。
2. 回答里的判断和概念原词。
3. 提问者提出的反驳、怀疑和追问。
4. 人物用来解释自己的例子和数字。

**压缩**：

1. 寒暄、重复确认、语气词、过场客套。
2. 过长的问题铺垫，只留转折那一句。
3. 不承载信息的问答标签。

**判断追问还是铺垫**：这句话是否改变了回答的方向，或让回答者重新组织了语言？是→保留；否→压缩或删除。问题本身有张力时，它就是页面证据，不要洗掉。

## 四、长文章材料

长文章 slides 要保留文章推进感。

优先抽取：

1. 作者怎样开场。
2. 哪些段落提供背景。
3. 哪些小标题改变讨论方向。
4. 哪些事实和例子支撑判断。
5. 文章最后停在什么话、什么问题、什么余味。

不要把小标题直接做成目录。小标题可以作为来源位置，页面标题要变成有信息量的笔记句。

## 五、页面内容类型

每页最好属于以下一种：

1. 原问题页：保留一个推动回答的问题。
2. 原话页：保留一句有判断密度的话。
3. 例子页：用一个故事或数字证明判断。
4. 对照页：呈现两个同时存在的视角。
5. 结构页：把材料里已经出现的结构排清楚。
6. 线索页：追踪一个反复出现的词。
7. 余味页：保留材料最后留下的问题或未完成方向。
8. 核心总结页：放在 deck 结尾，集中呈现最有获得感的嘉宾观点。

不要默认使用评论页。复盘页也要有材料锚点。

## 六、核心总结页

核心总结页是给读者最后带走的高密度笔记，不是生成者评论。写法接近用户给的长条笔记风格，但要进入 slides 结构。

要求：

1. 只写材料里嘉宾或作者已经表达过的观点。
2. 每条都要有获得感，包含判断、机制、数字、例子或方法中的至少两项。
3. 每条尽量具体，让不熟悉原文的人也能看懂。
4. 不追求固定条数，按材料含金量决定。通常 6 到 12 条。
5. 长访谈和完整播客默认拆成 2 到 5 页，每页 2 到 4 条。
6. 不加评论，不写路转粉、太精彩、值得学习这类外部感受。
7. 不用引号和破折号。严格原话可保留原意，但页面上优先转成干净转述。
8. 每条必须能回到来源锚点。写不出来源就删掉。

## 七、常见失败

1. 页面标题像文章导语。
2. 先写一个漂亮总结，再找材料支撑。
3. 把访谈问题全部洗掉，失去现场推进感。
4. 把材料重排成生成者自己的观点顺序。
5. 用本质、真正、最该带走的是这类词替材料发言。
6. 把数字和例子藏在副文里，主舞台只剩抽象句。
7. 结尾拔高，离开材料本身。
8. 核心总结变成读后感，只剩夸赞、评论和泛泛启发。
