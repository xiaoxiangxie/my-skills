# PPT 常用布局选择手册

本文档用于在生成 slides 前选择合适结构。原则是内容决定布局，不是为了变化而变化。

如果这套 slides 的目标是做读者笔记、播客复盘、人物稿梳理或长文摘要，优先选择能保留原句、语境、线索词和阅读推进感的版式，不优先选择高屋建瓴的判断页。

## 一、常用布局类型

| 布局 | 适合内容 | 不适合内容 |
|---|---|---|
| 标题页 | 开场、结尾 | 解释复杂信息、目录说明 |
| 标题加正文 | 一个判断加少量说明 | 多组并列信息 |
| 两列 | 对比、问答、观点加证据 | 三个以上并列点 |
| 三列 | 三个要点、三阶段、三原则 | 每列内容很长 |
| 上下结构 | 先抛结论，再展开解释 | 强对比内容 |
| 大数字 | 一个关键数据或规模感 | 多个互不相关数字 |
| 表格 | 参数、清单、分类、优先级 | 情绪表达、金句、排期 |
| 排期表 | 日程、里程碑、项目安排、拍摄计划 | 观点对比、人物金句 |
| 矩阵 | 两个维度下的分类判断 | 线性流程 |
| 时间线 | 历史、项目进度、访谈脉络 | 无时间顺序的观点 |
| 流程图 | 先后步骤、因果链条 | 并列观点 |
| 引用页 | 人物原话、价值观、转折句 | 信息密集解释 |
| 嘉宾观点页 | 访谈金句、嘉宾价值观、重要判断 | 纯事实罗列 |
| 图文页 | 图片作为证据或情绪 | 没有真实图片 |
| 图片网格 | 多个案例、现场、作品集 | 单一重点图片 |
| 图表加洞察 | 数据变化、趋势、结构占比 | 没有可信数据 |
| 对话页 | 访谈、冲突、问答 | 单人独白 |
| 总结页 | 收束、复盘、行动项 | 展开新论点 |
| 原文摘录加侧注 | 长文章、深度稿、值得摘录的原句 | 纯情绪页 |
| 段落拆解 | 一段高密度文字、需要拆层的段落 | 只有一句短话 |
| 线索词追踪 | 反复出现的关键词或主题词 | 一次性信息 |
| 观点加例证 | 一个重点配多个例子 | 纯抽象口号 |
| 一句原话，多层注解 | 一句关键原话需要深看 | 不重要的普通引语 |
| 文章脉络 | 交代长文或长谈如何推进 | 目录说明 |
| 双张力 | 两股力量同时成立 | 简单前后变化 |
| 未解问题 | 悬而未决的问题、余味、后续线索 | 强行收束的结论 |

## 二、内容到布局的选择规则

### 只有一个强判断

优先选择居中观点或标题页。不要加装饰元素。

### 一个判断需要解释

优先选择标题加正文或上下结构。标题在上，三条解释在下。

### 两个对象需要比较

优先选择两列或前后对比。左右两边结构要对称。

### 三个并列点

优先选择三列。每列一组标题和一句解释。

### 四个以上分类

优先选择表格、矩阵或图片网格。不要用大段列表。
表格用于清楚比较，不用于堆材料。默认使用淡底和行距分组，不用横线网格。排期不要用普通表格，使用排期表。

### 有时间顺序

优先选择时间线。节点不超过五个，超过五个就拆页。
如果内容是具体排期或 schedule，先判断它是时间线还是排期表。强调叙事变化用时间线，强调责任、状态、结果用排期表，二者不要混排。

### 有因果和步骤

优先选择流程图。每步用动词开头。

### 有人物原话

优先选择作者引用或对话页。必须显示说话者和语境。

如果这句原话本身已经足够重要，而且读者需要顺手记下它为什么重要，优先选择一句原话，多层注解或原文摘录加侧注。

### 有嘉宾观点

优先选择嘉宾观点页、作者引用或对话分裂。嘉宾观点页适合一句话特别重要，同时需要保留嘉宾身份和场景。

如果内容来自播客或访谈，而且重点在于这句话如何回应问题，优先选择一句原话，多层注解。

### 有数字

一个关键数字用大数字。多个同类数字用表格或图表加洞察。

### 有图片

一张强图优先用图文页或左右分屏。多张图用图片网格。

### 有一段高密度文字

优先选择段落拆解。先拆事实、重点、余味，再决定是否补引用页或结构页。

### 有反复出现的关键词

优先选择线索词追踪。不要把这种内容直接压成一句总结。

### 有一个重点和多个具体例子

优先选择观点加例证。比标题加正文更适合笔记型材料。

### 需要交代长文或长谈是怎么推进的

优先选择文章脉络。它不是目录页，重点是推进关系。

### 有两个同时成立、互相拉扯的重点

优先选择双张力。不要误用 before after。

### 结尾更适合保留问题

优先选择未解问题。不要急着做结论页。

## 四、信息密度规则

1. 轻页负责记忆点。
2. 信息页负责解释和可信度。
3. 每三页至少有一种明显的布局变化。
4. 同一种布局最多连续使用两页，且必须是系列内容。
5. 表格和矩阵页不能同时塞入长段正文。
6. 三列页每列不超过三行。
7. 时间线每个节点不超过两行。
8. 图表页必须有一句洞察，不只展示图表。
9. schedule 类页面必须统一字段和视觉样式，不要一部分像时间线，一部分像任务表。
10. schedule 超过六行必须拆页或合并阶段，不要压成小字号密表。
11. 播客复盘页数增加时，优先增加语境页、解释页、对话冲突页和复盘页，不要只增加大标题页。
12. 25 页以上的 deck 至少有三页明确属于复盘层，而不只是内容展开层。
13. 如果是长文章或人物稿，优先增加摘录侧注页、段落拆解页、线索词页和未解问题页，不要只增加观点页。

## 五、优先级

选择布局时按这个顺序判断：

1. 内容关系：并列、对比、时间、流程、分类、证据。
2. 信息密度：轻页、中密度、重信息。
3. 视觉节奏：上一页是什么结构，下一页需要怎样转场。
4. 演讲动作：宣告、解释、证明、停顿、反转、收束。

## 六、目录式页面处理

不要把章节结构直接做成页面内容。第一节、第二部分、接下来讲、这一页介绍等话术，只能存在于生成者的大纲里，不能出现在观众看到的 slide 正文中。

如果材料天然有章节，优先这样转译：

1. 章节名转译为主张。
2. 章节说明转译为一句证据。
3. 章节之间的衔接转译为转折判断。
4. 真正需要停顿时，使用转场判断页，不使用目录页。

## 七、笔记型材料的默认优先级

当材料本身是长文章、播客、访谈、人物稿或深度报道时，优先按这个顺序选布局：

1. 原文摘录加侧注
2. 段落拆解
3. 观点加例证
4. 一句原话，多层注解
5. 线索词追踪
6. 文章脉络
7. 双张力
8. 未解问题

也就是说，先把读者真正会记下来的内容立住，再补通用演讲页。
