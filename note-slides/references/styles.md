# 设计系统规范

本文档定义 slides skill 的视觉系统。目标不是做漂亮网页，而是做有舞台感的演示文稿。

## 一、整体气质

默认方向：克制、锋利、留白多、层级清楚。像发布会，也像杂志封面。观众应该先感到一句话的重量，再看到支撑它的证据。

### Jobs 式原则

1. 一页只讲一个想法，但要给这个想法足够的支撑。
2. 字要大，词要准，信息要分层。
3. 空白是权威感的一部分，也是组织信息的工具。
4. 图片不是装饰，是证据或情绪。
5. 动效只服务翻页节奏，不炫技。
6. 复杂内容拆页，但每页都要有完整的信息闭环。
7. 样式丰富不等于每页换皮。真正的丰富来自内容关系不同，视觉系统仍然一致。
8. 先判断证据核心，再判断版式。数字、原话、图片和界面证据都有自己的天然舞台。

## 二、字体系统

所有字号必须用 `clamp()`。1920×1080 画布下，正文视觉尺寸不低于 24px。

| 用途 | class | 字体族 | 字重 | clamp 范围 |
|---|---|---|---|---|
| 封面大标题 | `.display-zh` | var(--serif-zh) | 900 | `clamp(3rem, 7vw, 7rem)` |
| 页面主标题 | `.h1-zh` | var(--serif-zh) | 700 | `clamp(2rem, 4.6vw, 4.2rem)` |
| 页面副标题 | `.h2-zh` | var(--serif-zh) | 600 | `clamp(1.5rem, 3vw, 2.6rem)` |
| 引导段 | `.lead-zh` | var(--serif-zh) | 400 | `clamp(1.5rem, 2vw, 2rem)` |
| 正文 | `.body-zh` | var(--sans-zh) | 400 | `clamp(1.5rem, 1.55vw, 1.75rem)` |
| 小节标签 | `.kicker` | var(--mono) | 400 | `clamp(.8rem, .9vw, 1rem)` |
| 元信息 | `.meta` | var(--mono) | 400 | `clamp(.75rem, .8vw, .9rem)` |

### 字体变量

```css
--serif-zh: "Noto Serif SC", "Source Serif 4", serif;
--sans-zh: "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;
--mono: "IBM Plex Mono", monospace;
```

### 字体使用规则

1. 标题和金句用衬线字体，制造重量。
2. 正文用非衬线字体，保证快速扫描。
3. 页码、章节、来源用等宽字体，提供冷静秩序。
4. 同一页最多三种字号。
5. 正文一行尽量不超过 16 个汉字，标题可以更短更猛。
6. 信息页允许小一号注释，但注释必须用于来源、语境或转折。

## 三、配色系统

note-slides 默认色彩是一套冷纸、近黑、清蓝系统。它适合访谈、商业、科技、产品、组织和方法论内容。暖纸、深墨、暗红保留为纸本纪实方向，不再作为所有内容的自动答案。

| 变量名 | 色值 | 用途 |
|---|---|---|
| `--ink` | `#0d1117` | 深色观点页、封面和总结页 |
| `--paper` | `#f5f7fa` | 浅色解释页和证据页 |
| `--accent` | `#1869d4` | 关键数字、页内编号、观点锚点 |
| `--muted` | `#636c7a` | 次级信息和页眉页脚 |

### 用色规则

1. 一页只允许一个强调色重心。
2. 不用彩虹色，不用装饰渐变。
3. 强调色只用于关键字、页内唯一数字或章节标记。
4. 深色页用于观点和情绪，浅色页用于解释和证据。
5. 每两到三页切换主题，禁止连续三页同主题。

### 配色推导流程

做每套 slides 前，先写一句 `palette thesis`，解释这组颜色为什么属于这篇内容。

至少从这五个维度判断：

1. 情绪温度：这篇内容是温热、冷静、锋利、忧郁、纪实，还是混合状态。
2. 人物气质：说话者是摇滚、知识分子、商业发布、技术系统、民间叙事，还是别的类型。
3. 时代和材质：更像纸本访谈、旧杂志、舞台灯光、屏幕 UI、档案页、实验室，还是别的媒介感。
4. 冲突类型：这篇内容是观点碰撞、回忆追述、技术焦虑、公共议题，还是人物剖面。
5. 与上一套的差异：题材不同、人物不同、时代感不同，就要显式说明这次颜色关系怎么变。

### 什么时候不能直接用默认冷静蓝黑

以下情况默认冷静蓝黑可能不够，需要重推：

1. 材料强烈依赖人物记忆、文学、音乐、地方经验或纸本气质。
2. 主题有明显热度、现场感、舞台感或怀旧感，冷色会削弱人物气味。
3. 内容不是访谈笔记，而是品牌页、产品展示、视觉作品集或活动海报。
4. 和上一套 deck 的叙事气质明显不同。

### 常用色调方向

不要把这部分当模板库，而是当判断参考：

1. 访谈、商业、科技、产品、组织和方法论：优先使用冷纸、近黑、清蓝系统。
2. 人物回忆、纪实、文学、音乐：可以用暖纸、深墨、暗红或烟草色系。
3. 技术、AI、系统变迁、未来焦虑：可以在冷静蓝黑基础上提高黑场比例，强调色更克制或更电性。
4. 商业、策略、汇报、结构化判断：优先考虑更中性的背景和更清楚的高对比强调色。
5. 温柔叙事、关系、日常、教育：可以保留暖度，但降低攻击性，减少高饱和强调。

### 可选美学方向

每套 slides 至少选择一种主方向，并允许少量混合。不要每次都回到暖纸、深墨、暗红。

| 方向 | 适合内容 | 视觉关键词 | 配色倾向 |
|---|---|---|---|
| 冷静访谈笔记 | 创始人访谈、商业思考、科技人物、长篇问答 | 黑场封面、冷纸证据页、清蓝编号、等宽页眉页脚 | 近黑、冷白、清蓝、冷灰 |
| 纸本纪实 | 人物访谈、回忆、文学、音乐 | 旧杂志、长留白、衬线重量 | 暖纸、深墨、暗红、烟草 |
| 冷静系统 | AI、技术、产品、组织变化 | 冷白、硬边界、精确间距 | 冷灰纸、近黑、蓝绿或电青 |
| 发布会 | 商业判断、策略、产品叙事 | 高对比、强标题、舞台感 | 深底、亮纸、单一高亮色 |
| 档案室 | 历史、调查、资料梳理 | 编号、索引、低饱和、证据感 | 灰纸、炭黑、棕灰或暗橄榄 |
| 黑胶舞台 | 音乐、现场、人物气质强的内容 | 大黑场、聚光、强引用 | 黑、米白、红或金属色 |
| 清透教育 | 课程、方法论、复盘 | 清楚、轻、可扫描 | 浅底、深字、低饱和强调 |
| 深界面编辑 | 产品 UI、系统方案、工具链、开发者叙事 | 黑场、细编号、左侧论点、右侧界面证据 | 近黑、骨白、冷灰、低饱和强调 |

选择方向后，整套只保留一套主视觉语法。可以换布局，但页眉、页脚、表格、编号、强调色的处理必须一致。

### 冷静访谈笔记规则

这类方向是 note-slides 的默认主系统，适合用户认可的创始人访谈和商业技术访谈成品。
`template.html` 已经按这套成品语言提供基础页型。生成新 deck 时优先复用模板里的页型和类名，不要重新发明一套外观。

1. 全套使用统一的页眉和页脚。左侧放来源或章节语境，右侧放页码、NOTE SLIDES 或材料类型，透明度低，不抢内容。
2. 深色页承担封面、嘉宾判断、强问题、未解问题和核心总结。浅色页承担语境、证据、表格、时间线和拆解。
3. 版式数量要少，但可以用同一骨架做轻微语义变化。不要为了显得丰富而每页换一套装饰。
4. 大标题用衬线字体，编号、页眉、页脚和标签用等宽字体。编号本身可以成为画面锚点。
5. 页面尽量裸排，少用卡片。需要分组时用留白、细线、编号和对齐关系解决。
6. 不使用渐变、图标、装饰图形和无证据图片。没有真实视觉证据时，宁可保持文字系统干净。
7. 三列页、左右页、表格页都要让主张和证据形成一个视觉组，避免标题漂在上方、内容沉在底部。
8. 核心总结页使用深色底、大号编号和长条笔记，和前面的证据页形成收束。

### 输出要求

在进入 HTML 之前，至少明确这四个变量的角色：

1. `--ink`：主舞台是厚重、冷硬、安静还是明亮。
2. `--paper`：解释页是暖纸、冷白、灰纸还是别的中性底。
3. `--accent`：强调色是警示、情绪、章节锚点，还是数据重心。
4. `--muted`：次级信息是靠褪色、发灰还是降饱和来退后。

## 四、留白和构图

### 舞台边距

```css
padding: clamp(2rem, 5vw, 5.5rem) clamp(2rem, 6vw, 6.5rem);
```

### 构图规则

1. 默认把主张放在画面中心或视觉中心附近。
2. 中央构图用于宣告、章节、引用和结论。
3. 左右分屏用于观点加证据、过去加现在、人物加观点。
4. 上下分层用于先抛判断，再给三条解释。
5. 页眉页脚只做定位，不抢内容。
6. 卡片感要少。重复信息可以并列，不要包成一堆容器。
7. 大标题周围要有完整呼吸区。
8. 居中大标题最多两行。超过两行说明这不是宣告页，应该改成左右分屏、表格、时间线或拆页。
9. 人物履历、年份、增长、产品范围不能和一个超大居中句子同页竞争。履历页用清楚的结构，不用海报式大标题加仪表盘。
10. 同一页不要把不可比的信息做成同等重量的三个视觉锚点。比例、时长、产品列表、人物履历属于不同信息类型，应该分层处理。
11. 内容区禁止右对齐。中文标题、正文、引用、双列对比默认只用左对齐或居中。右对齐只允许用于页眉页脚右侧、页码、表格数字列或极小的辅助标签。
12. 左右分屏不是左右贴边。两列都要作为一个整体落在视觉中心，右列也不能把文字右对齐。
13. 标题加三列、标题加表格、标题加矩阵必须作为一个整体垂直居中。不要让三列网格 `flex:1` 撑满剩余空间。
14. `.grid-3` 和 `.grid-6` 默认不写 `flex:1`。只有整页主结构确实需要占满高度时，才在外层容器单独控制。
15. 三列页不在左下角、右下角或页脚上方另放长补充段。补充段会变成第四个内容块，破坏三列结构。
16. 如果三列之外还有一段重要说明，把它合并进某一列，改成表格，或拆成下一页。内容区的悬浮脚注不超过 18 个中文字符，只写来源或极短语境。
17. 三列页要克制使用。只有三列确实是同类并列，且每列标题同语法、正文两到三行时才成立。看到连续的标题加三列页面，要优先改成摘录侧注、段落拆解、观点加例证、表格、时间线或双张力。

### 构图配比

一套 slides 里，建议采用以下比例：

1. 居中宣告类约 30%。
2. 分屏解释类约 30%。
3. 引用和人物语境类约 15%。
4. 数据、时间线、流程类约 15%。
5. 全图或情绪停顿类约 10%。

其中标题加三列的 L9 不单独承担 30% 的解释类份额。它通常只占全套 0 到 12%，其余解释页由左右分屏、段落拆解、摘录侧注、表格、时间线和观点加例证承担。

### 版式风格变体

同一套 deck 可以有多种版式，但只能有一种主视觉语法。变化来自构图关系，不来自每页换一套装饰。

1. 社论大标题：适合强判断，标题占主视觉，副文只补最小语境。
2. 证据索引：适合事实密集页，用编号、来源、短注释组织可信度。
3. 舞台分屏：适合人物、冲突、对话，一侧压暗，一侧承担主声音。
4. 档案页：适合历史和资料，用等宽标签、低饱和淡底、紧凑说明。
5. 数据停顿：适合一个关键数字，数字本身是画面重心。
6. 排期页：适合 schedule 和里程碑，用统一表格系统，不和时间线混用。
7. 全图停顿：适合有真实图片证据时使用，文字必须短，图片不能只是气氛。
8. 界面证据页：适合产品、开发工具和系统方案。左侧用大标题和三到四条证据行，右侧放真实界面截图或低调占位。

### 避免 AI 味视觉捷径

1. 不用大面积渐变制造高级感。
2. 不用无意义图标、假数据、装饰 SVG 或过度圆角卡片填空间。
3. 不把每页做成网页 hero、功能卡片或后台仪表盘。
4. 不复制参考图的可识别版式，只提炼层级、留白、材质和阅读路径。
5. 字体选择要符合内容气质，不默认使用过度常见的产品 UI 字体。

### 深界面编辑规则

这类风格参考黑场编辑页，但不复制任何具体设计。它的价值在于把抽象方案和可见界面证据放在同一页。

1. 使用近黑背景和骨白文字，整体对比强，但次级信息要明显退后。
2. 页眉、页码、页脚使用等宽字、宽字距和低透明度，形成冷静索引感。
3. 标题可以很大，允许中英混排，但只服务一个主张。
4. 左侧证据列表最多四行，每行包含标签、解释和一个状态词。
5. 右侧必须是截图、产品面板、线框或真实证据，不要放装饰插画。
6. 细分割线只用于证据行，不得变成整页横线装饰。
7. 如果没有真实截图，用 `interface-placeholder` 做克制占位，并在之后提醒需要替换。

## 五、内容密度

### 每页上限

1. 一个主标题
2. 一到三条解释或一个辅助句
3. 一个证据元素，可以是引用、数字、图片、对比、时间线
4. 一个来源、作者或语境标记
5. 一个页脚定位
6. 如果主舞台是原话，原话中文不超过 36 个字。超过 36 个字时，这页只能做独立引用或拆成两页。
7. 原话页下方最多放一个短语境，不再接三列解释。

### 强制拆页

出现以下情况必须拆成多页：

1. 正文超过四行。
2. 列表超过四项。
3. 同时出现引用、图片和数据。
4. 一个页面有两个互相竞争的判断。
5. 上方大引用超过三行，同时下方还有三列、表格或长解释。
6. 一页既想保留长原话，又想解释为什么、补事实、补背景。长原话独立成页，解释另起一页。

### 信息闭环

每页都要回答三个问题中的至少两个：

1. 这页在说什么。
2. 为什么可信。
3. 和上一页或下一页有什么关系。

### 观众阅读路径

每页必须能说清楚三步：

1. 第一眼看到什么。
2. 第二眼理解什么。
3. 第三眼用什么证据相信它。

如果三步路径不清楚，优先调整层级，不要先调颜色。

## 六、图片规则

1. 真实图片优先。
2. 图片必须用 `<figure class="slide-img">` 包裹。
3. 图片高度用 `height:clamp(...vh...)` 控制，禁止用 aspect-ratio。
4. `object-position: top center`，只裁底部。
5. 人物图不能裁头、眼睛、手势。
6. 没有图时，用低调占位，不要画复杂 SVG 插画。
7. 图片说明简短，只写来源或语境。

推荐高度：

| 场景 | 高度 |
|---|---|
| 辅助图 | `clamp(24vh, 34vh, 44vh)` |
| 主图 | `clamp(36vh, 54vh, 68vh)` |
| 图像网格 | `clamp(18vh, 24vh, 30vh)` |

## 七、引用规则

引用是节拍点，也是一种证据。最多每四到五页出现一次。

1. 只保留最锋利的句子。
2. 超过三行就拆。
3. 来源必须简短。
4. 引用页必须有作者、身份或场景。
5. 引用页周围留白要更大。
6. 左边框可以保留，但不要再叠加圆角卡片、阴影或图标。
7. 大号引用最多 36 个中文字符，英文最多 22 个词。更长的内容改成段落摘录页，字号降级，或拆成两页。
8. 大号引用和三列解释不能同页出现。三列页的上方标题只能是压缩后的短判断。

## 八、数据规则

数据只在能改变判断时出现。

1. 一页最多一个主数字或一组强关联数字。
2. 数字必须有一句解释。
3. 数字必须有来源或上下文。
4. 不要为了显得专业而添加无意义指标。

## 九、动效规则

动效应像灯光亮起，而不是网页进入动画。

1. 标题慢一点，正文快一点。
2. 每页最多三组入场。
3. 不给纯布局容器加动画。
4. 不使用旋转、弹跳、复杂路径。
5. 翻页节奏优先于单页动效。

## 十、组件审美

1. 少用卡片。
2. 少用图标。
3. 少用圆角。
4. 少用阴影。
5. 禁止用横向分割线作为装饰或分组手段。
6. 表格可以使用极淡底色或行距区分，不用横线网格。
7. 三列、矩阵、时间线不使用贯穿页面的横线。
8. 需要分组时，用位置、字号、编号、留白和淡底，而不是线条。

## 十一、表格规则

表格是信息设计，不是网页控件。它应该像一张干净的编辑页，而不是后台管理系统。

1. 默认使用 `table-wrap` 和 `data-table` 类。
2. 默认不使用横线网格，不使用粗边框，不使用阴影。
3. 表头用等宽小字和极淡强调底，正文靠行距和淡底分组。
4. 默认整表居中。只有第一列承载行名、时间或对象时，给表格加 `keyed` 类，让第一列左对齐并加字重。
5. 如果单元格是长句，给表格加 `left` 类，整表左对齐。
6. 普通表格超过四行或四列，优先拆页，或者改成矩阵、流程、时间线。
7. 排期、schedule、里程碑表使用排期表，三到五行最稳，超过六行必须拆页或合并为阶段。
8. 排期表必须统一列语义，不要混合日期、任务、备注、观点四种不对等内容。
9. 表格标题和表格作为一个整体居中，宽度默认 `min(100%, 74rem)`，除非高密度信息确实需要更宽。
10. 新 deck 优先直接复制 `template.html` 的 Table 页型。不要自己临时写 `table` 样式，不要通栏铺满，不要用后台管理系统式边框和阴影。
11. 表格外层必须放在 `slide-body` 内，标题和表格必须作为一个 `table-wrap` 组垂直居中。
